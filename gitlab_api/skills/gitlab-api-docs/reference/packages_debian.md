# Debian API
  * Tier: Free, Premium, Ultimate
  * Offering: GitLab Self-Managed


History
  * [Deployed behind a feature flag](https://docs.gitlab.com/administration/feature_flags/), disabled by default.


This API is used by the Debian related package clients such as [dput](https://manpages.debian.org/stable/dput-ng/dput.1.en.html) and [apt-get](https://manpages.debian.org/stable/apt/apt-get.8.en.html), and is generally not meant for manual consumption. This API is under development and is not ready for production use due to limited functionality.
Use this API to interact with the [Debian package manager client](https://docs.gitlab.com/user/packages/debian_repository/).
These endpoints do not adhere to the standard API authentication methods. See the [Debian registry documentation](https://docs.gitlab.com/user/packages/debian_repository/) for details on which headers and token types are supported. Undocumented authentication methods might be removed in the future.
## Enable the Debian API[](https://docs.gitlab.com/api/packages/debian/#enable-the-debian-api "Permalink")
The Debian API is behind a feature flag that is disabled by default. [GitLab administrators with access to the GitLab Rails console](https://docs.gitlab.com/administration/feature_flags/) can opt to enable it. To enable it, follow the instructions in [Enable the Debian API](https://docs.gitlab.com/user/packages/debian_repository/#enable-the-debian-api).
## Enable the Debian group API[](https://docs.gitlab.com/api/packages/debian/#enable-the-debian-group-api "Permalink")
The Debian group API is behind a feature flag that is disabled by default. [GitLab administrators with access to the GitLab Rails console](https://docs.gitlab.com/administration/feature_flags/) can opt to enable it. To enable it, follow the instructions in [Enable the Debian group API](https://docs.gitlab.com/user/packages/debian_repository/#enable-the-debian-group-api).
### Authenticate to the Debian Package Repositories[](https://docs.gitlab.com/api/packages/debian/#authenticate-to-the-debian-package-repositories "Permalink")
See [Authenticate to the Debian Package Repositories](https://docs.gitlab.com/user/packages/debian_repository/#authenticate-to-the-debian-package-repositories).
## Upload a package file[](https://docs.gitlab.com/api/packages/debian/#upload-a-package-file "Permalink")
Uploads a Debian package file for a specified project.
```
PUT projects/:id/packages/debian/:file_name
```

Attribute | Type | Required | Description
---|---|---|---
`id` | string | yes | The ID or full path of the project.
`file_name` | string | yes | The name of the Debian package file.
`distribution` | string | no | The distribution codename or suite. Used with `component` for upload with explicit distribution and component.
`component` | string | no | The package file component. Used with `distribution` for upload with explicit distribution and component.
shell
```
curl --request PUT \
     --user "<username>:<personal_access_token>" \
     --upload-file path/to/mypkg.deb \
     --url "https://gitlab.example.com/api/v4/projects/1/packages/debian/mypkg.deb"
```

Upload with explicit distribution and component:
shell
```
curl --request PUT \
  --user "<username>:<personal_access_token>" \
  --upload-file  /path/to/myother.deb \
  --url "https://gitlab.example.com/api/v4/projects/1/packages/debian/myother.deb?distribution=sid&component=main"
```

## Download a package[](https://docs.gitlab.com/api/packages/debian/#download-a-package "Permalink")
Downloads a specified package file for a project.
```
GET projects/:id/packages/debian/pool/:distribution/:letter/:package_name/:package_version/:file_name
```

Attribute | Type | Required | Description
---|---|---|---
`distribution` | string | yes | The codename or suite of the Debian distribution.
`letter` | string | yes | The Debian Classification (first-letter or lib-first-letter).
`package_name` | string | yes | The source package name.
`package_version` | string | yes | The source package version.
`file_name` | string | yes | The filename.
shell
```
curl --header "PRIVATE-TOKEN: <personal_access_token>" \
  --url "https://gitlab.example.com/api/v4/projects/1/packages/debian/pool/my-distro/a/my-pkg/1.0.0/example_1.0.0~alpha2_amd64.deb"
```

Write the output to a file:
shell
```
curl --header "PRIVATE-TOKEN: <personal_access_token>" \
     --url "https://gitlab.example.com/api/v4/projects/1/packages/debian/pool/my-distro/a/my-pkg/1.0.0/example_1.0.0~alpha2_amd64.deb" \
     --remote-name
```

This writes the downloaded file using the remote filename in the current directory.
## Route prefix[](https://docs.gitlab.com/api/packages/debian/#route-prefix "Permalink")
The remaining endpoints described are two sets of identical routes that each make requests in different scopes:
  * Use the project-level prefix to make requests in a single project’s scope.
  * Use the group-level prefix to make requests in a single group’s scope.


The examples in this document all use the project-level prefix.
### Project-level[](https://docs.gitlab.com/api/packages/debian/#project-level "Permalink")
```
/projects/:id/packages/debian
```

Attribute | Type | Required | Description
---|---|---|---
`id` | string | yes | The project ID or full project path.
### Group-level[](https://docs.gitlab.com/api/packages/debian/#group-level "Permalink")
```
/groups/:id/-/packages/debian
```

Attribute | Type | Required | Description
---|---|---|---
`id` | string | yes | The project ID or full group path.
